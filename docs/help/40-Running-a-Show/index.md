# Running a Show

Running a show means executing stored Cues and masters while maintaining clear ownership between the programmer, Preload, and playbacks.

![Cuelist detail and the configured playback bank](../assets/screenshots/cuelist-playback.png)

Before operation, assign every required target, verify playback pages and screen page modes, clear diagnostic overrides, release temporary programmers, and run the complete Cue sequence.

Continue with [Cues and Playbacks](01-cues-and-playbacks.md), [HTP, LTP, and Ownership](02-htp-ltp-and-ownership.md), [Preload and Preload GO](03-preload.md), and [Triggers, Chasers, and Speed Groups](04-triggers-chasers-and-speed.md).
